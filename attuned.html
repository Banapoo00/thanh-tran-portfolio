Overview

Attuned is built on a single insight from parent research: parents don't lack knowledge about how to respond to their child. They lose access to it when they're flooded with stress. The product is built to regulate the parent first — because a regulated parent is the one who can actually help their child.

The product spans four core experiences. I joined as the AI PM for Know My Child — the feature that builds a living, evolving portrait of a child across every Coa conversation, getting more specific over time rather than staying generic. I invite you to read about the product launch for Know My Child in depth below.

AI Model Overview
Design Principle

Invest in proprietary data and retrieval layers, not model fine-tuning. The real moat is our proprietary approach and the parent memory that compounds over time — not the AI model itself.

Extraction
gpt-5-nano

Purpose: pulls structured JSON — behavior, emotion, and need candidates — from the parent's raw message. Temperature = 0.

Characteristics: fast, cheap, deterministic. Runs before the parent sees anything.

Generation
GPT-5.4 mini

Purpose: generates every Coa response — framework-grounded replies, pattern insights, bedtime stories — each with its own prompt.

Characteristics: receives extracted JSON + retrieved context from our proprietary framework + child profile; produces three risk-tiered candidate responses per message.

Embeddings
text-embedding-3-small

Purpose: converts our proprietary reference material, session summaries, and parent memory into vectors for semantic retrieval.

Characteristics: powers retrieval across every feature via pgvector, stored in Supabase Postgres.

Safety Judge
gpt-5-nano

Purpose: reviews each candidate against a clinical safety rubric — diagnosis, treatment claims, unsupported causal claims. Evaluates only, never generates.

Characteristics: a binary classification task, so it adds minimal latency to the response pipeline.

Retrieval (RAG)
pgvector, within Supabase Postgres

Purpose: retrieves the closest matching context via semantic similarity, filtered by the child's age band, and pulls per-parent session history to power pattern synthesis over time.

Characteristics: lives inside the existing Postgres database — no separate vector store to run or maintain. This is the layer that makes Coa feel like it remembers a family, not just a single message.

Why Two Models

gpt-5-nano handles extraction and safety judging; GPT-5.4 mini is reserved only for generation — the one task that actually needs it.

Both extraction and safety judging are fast, cheap, deterministic tasks — well suited to a smaller model. Generation is where quality matters most, so that's the only place the more expensive model is used. Because intelligence lives in the retrieval layer, not the model, either model can be swapped for an open-source alternative (Mistral, Llama 4) at scale without rebuilding anything.

User Stories & Acceptance Criteria
Category
User Story
Acceptance Criteria
Portrait Synthesis
As a parent, I want to see a warm portrait of who my child is right now, so that I can understand their behaviour without feeling blamed or judged.
Portrait is 80–120 words, written in Coa's voice, includes at least one positive trait, and ends with a normalising forward-looking statement. No clinical language. Renders within 3 seconds of tab open.
Pattern Cards
As a parent, I want to see the patterns behind my child's hard moments, so that the same situations start to make sense instead of feeling random.
Exactly 2 pattern cards. Each includes a plain-language label (3–6 words), a single approved need tag, a one-sentence explanation (max 20 words), and a micro-action. Patterns must be genuinely distinct from each other.
What's Been Helping
As a parent, I want to know what I've been doing that's helping my child, so that I feel my effort is being noticed and I'm encouraged to keep going.
Insight only generates after ≥2 Tried It signals. Insight is 30–50 words, confirmatory (what parent has done) not prescriptive (what to do next). Warm placeholder renders when threshold not met.
Feedback Signals
As a parent, I want to give feedback on whether a pattern resonates, so that future portraits feel increasingly accurate to my specific child.
This Resonates and Not Right Now signals log to the database immediately on tap — not deferred. Not Right Now patterns excluded from next synthesis cycle. Back button after This Resonates returns parent to read-only card.
Unlock & Analytics
As a parent, I want Know My Child to unlock naturally after enough sessions, so the insights feel earned rather than arbitrary.
know_my_child_unlocked PostHog event fires exactly once per child profile at session 3. All P0 analytics events verified in PostHog live stream before ticket closes. No PII sent to PostHog.
Risk
High
AI output tone
Synthesis could produce clinical, diagnostic, or shame-inducing content about a real child.
→ Output guardrail prompt reviews every component before it reaches the parent. Retry and fallback logic built in.
High
Synthesis failure
Agentic workflow fails silently — parent opens tab and sees broken or empty content.
→ Fallback to last successful synthesis. If none exists, warm waiting state renders. Parent never sees an error screen.
Medium
Low session volume
Parents don't reach the 3-session unlock, so Know My Child never surfaces.
→ Lock screen designed to preview what unlocks. Progress bar shows momentum. Unlock celebration rewards reaching threshold.
Medium
Portrait feels generic
Synthesis produces output that doesn't feel specific to this child — destroying trust.
→ Child profile + family context + session history all passed into Prompt 1. Low temperature (0.3) keeps output grounded.
Low
Pattern card duplication
Two pattern cards surface the same underlying theme — eroding credibility.
→ patterns_are_distinct validation in Prompt 2. Model must revise until both patterns are genuinely distinct.
Low
Cost scaling
3 synthesis prompts per child per cycle could become costly at scale.
→ Prompt caching on system roles reduces input costs 75–90%. Synthesis only triggers per child profile, not per session.
